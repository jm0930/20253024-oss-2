## 1. top이란?

`top`은 리눅스의 “작업 관리자” 같은 명령어입니다.  
실행 중인 프로그램들이 CPU나 메모리를 얼마나 쓰는지 **실시간으로 보여줍니다.**

---
##사용법
top # 실행
P # CPU 정렬
M # 메모리 정렬
k # 프로세스 종료
q # top 종료

---

## 화면에서 보이는 정보

- CPU 사용률  
- 메모리 사용량  
- 실행 중인 프로세스 목록  
- 프로세스 ID(PID)  

위쪽은 시스템 전체 요약,  
아래쪽은 프로그램 목록이라고 보면 됩니다.

---

## 자주 쓰는 키

| 키 | 기능 |
|----|------|
| P | CPU 사용량 기준 정렬 |
| M | 메모리 사용량 기준 정렬 |
| k | 프로세스 종료(PID 입력) |
| q | top 종료 |

---

## 예시

- CPU 많이 쓰는 프로그램 보기  
top → P
- 메모리 많이 쓰는 프로그램 보기  
top → M
- 특정 프로그램 종료  
top → k → PID 입력

---

## 2. ps란?

`ps`는 현재 실행 중인 프로세스를 **한 번에 출력해주는 명령어**입니다.  
top처럼 실시간은 아니고, **딱 실행한 순간의 상태(Snapshot)** 을 보여줍니다.

---

## 사용법
ps # 기본
ps -e # 전체 프로세스 보기
ps aux # 상세하게 보기

---

## 화면에서 보이는 정보

- 실행 중인 프로세스 목록  
- PID(프로세스 ID)  
- USER(누가 실행했는지)  
- CPU/메모리를 얼마나 쓰는지  
- COMMAND(실행 중인 프로그램 이름)  

---

## 자주 쓰는 키

| 옵션 | 설명 |
|------|------|
| -e | 시스템 전체 프로세스 보기 |
| aux | 모든 사용자 + 자세한 정보 표시 |
| -f | full format(계층/부모 PID까지 표시) |

---

## 예시

- 전체 프로세스 보기  
ps -e
- 자세히 보기 (가장 많이 사용함)  
ps aux
- 특정 프로그램 찾기 (grep과 함께 사용)  
ps aux | grep python

---

## 3. jobs란?

`jobs`는 **현재 터미널에서 실행 중인 작업 목록**을 보여주는 명령어입니다.  
리눅스는 명령어 뒤에 `&` 를 붙이면 백그라운드로 실행되는데,  
jobs는 이런 작업들의 상태를 확인할 때 사용합니다.

---

## 사용법
jobs       # 현재 터미널의 작업 목록 보기

---

## 화면에서 보이는 정보
- 작업 번호(Job ID)
- 실행 상태 (Running / Stopped)
- 실행 중인 명령어(COMMAND)
- 현재 터미널(셸) 기준으로만 표시됨

---

## 자주 쓰는 키
| 옵션 | 설명 |
|------|------|
| -l | PID까지 함께 출력 |
| -n | 최근에 상태가 변경된 작업만 표시 |
| -p | PID만 출력 |

---

## 예시

백그라운드 실행
sleep 5 &

작업 목록 확인
jobs

중지된 작업 확인 (Ctrl + Z 후)
jobs
